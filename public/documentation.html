Documentation Page

This is a API that provides users with access to a movie database where they could search about different movies, directors, and genres. Users will be able to sign up, update their personal information, and create a list of their favorite movies.

GET
"/movies"
-> Return a list of ALL movies to the user

Inputs

Parameters: "None"
Body: "None"

Response

"A JSON object holding data about all movies."
Ex.: [ { "genre": { "name": "Thriller", "description": "Thriller film, also know as..." }, "director": { "name": "Jonathan Demme", "bio": "Robert Jonathan Demme was an American...", "birth":"1944", "death": "2017" }, "actors": [], "_id": "62704010caaa0abffd6491ea", "title": "Silence of the Lambs", "description": "updated", "imagePath": "silenceofthelambs.png", "featured": true }, ]"

GET
"/movies/:title"
-> Return data (description, genre, director, image URL, whether it's featured or not) about a single movie by title to the user

Inputs

Parameters: "title"
Body: "None"

Response

"A JSON object holding all data about the searched movie."
Ex.: { "genre": { "name": "Fantasy", "description": "The Fantasy genre is..." }, "director": { "name": "Tim Burton", "bio": "Tim Burton was born in ...", "birthyear": "some date", "deathyear": "still a live" }, "actors": [], "_id": "62711e29feca33c02c3251e9", "title": "Beetlejuice", "description": "A couple of recently deceased ghosts contract the services of...", "imageUrl": "https://images-na.ssl-images-amazon.com/images/M/MV5BMTUwODE3MDE0MV5BMl5BanBnXkFtZTgwNTk1MjI4MzE@._V1_SX300.jpg", "featured": true }"

GET
"/genre/:name"
-> Return data about a genre (description) by name/title (e.g., “Thriller”)

Inputs

Parameters: "name"
Body: "None"

Response

"A JSON object holding all infos about the genre."
Ex.: { "name": "Fantasy", "description": "The Fantasy genre is..." }"

GET
"/directors/:name"
-> Return data about a director (bio, birth year, death year) by name

Inputs

Parameters: "name"
Body: "None"

Response

"A JSON object holding all data about the searched director."
Ex.: { "name": "Tim Burton", "bio": "Tim Burton was born in ...", "birthyear": "some date", "deathyear": "still a live" }"

POST
"/users"
-> Allow new users to register

Inputs

Parameters: "None"
Body: "username": "Lulu", "password": "12344", "email": "lulu@lulu.com", "birthday": "1999-04-05"

Response

"A JSON object holding data about the user that was added, including an ID."
Ex.:{ ID: 1, "username": "Lulu", "password": "12344", "email": "lulu@lulu.com", "birthday": "1999-04-05" }"

GET
"/users/:username"
-> Get user by username

Inputs

Parameters: "username"
Body: "None"

Response

"A JSON object holding data about the user, including an ID."
Ex.:{ ID: 1, "username": "Lulu", "password": "12344", "email": "lulu@lulu.com", "birthday": "1999-04-05" }"

POST
"/login"
-> Allow users to login

Inputs

**Choose between Parameters or Body to login**
Parameters: "username, password"
Body: "username": "Lulu", "password": "12344"

Response

"A JSON object holding data about the user that was loged, including an ID and a autorization token."
Ex.:{ ID: 1, "username": "Lulu", "password": "12344", "email": "lulu@lulu.com", "birthday": "1999-04-05" } {"token": "dslfaihfoiahodfhaljdfkasd..."}"

PUT
"/users/:username"
-> Allow users to update their user info (username)

Inputs

Parameters: "username"
Body: "None"

Response

"A JSON object holding data about the user that was updated."
Ex.: "{Email: lulu_Updated@lulu.com}"

POST
"/users/:username/movies/:movieID"
-> Allow users to add a movie to their list of favorites (showing only a text that a movie has been added—more on this later)

Inputs

Parameters: "username, movieID"
Body: "None"

Response

"A JSON object holding all data about the user, including the added movie."
Ex.: "{ ID: 1, username: "Lulu", password: 12344, email: lulu@lulu.com, birthday: 1999-04-05, favoriteMovies: ["62711f4dfeca33c02c3251ed" ] }"

DELETE
"/users/:username/movies/:movieID"
-> Allow users to remove a movie from their list of favorites (showing only a text that a movie has been removed—more on this later)

Inputs

Parameters: "username, movieID"
Body: "None"

Response

"A JSON object holding all data about the user, excluding the added movie."
Ex.: "{ ID: 1, username: "Lulu", password: 12344, email: lulu@lulu.com, birthday: 1999-04-05, favoriteMovies: [] }"

DELETE
"/users/:username"
-> Allow existing users to deregister (showing only a text that a user email has been removed—more on this later)

Inputs

Parameters: "username"
Body: "None"

Response

"A text message indicating whether the user was deleted"
Ex.: "None"
