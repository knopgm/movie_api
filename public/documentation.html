Documentation Page

This is a API that provides users with access to a movie database where they could search about different movies, directors, and genres. Users will be able to sign up, update their personal information, and create a list of their favorite movies.

  • Return a list of ALL movies to the user

    URL: "/movies"

    Method: GET

    Query parameters: "None"

    Request: "None"

    Response: "A JSON object holding data about all movies"

  • Return data (description, genre, director, image URL, whether it's featured or not) about a single movie by title to the user

    URL: "/movies/:title"

    Method: GET

    Query parameters: "title"

    Request: "None"

    Response: "A JSON object holding all data about the searched movie"

  • Return data about a genre (description) by name/title (e.g., “Thriller”)

    URL: "/movies/genre/:genreName"

    Method: GET

    Query parameters: "genreName"

    Request: "None"

    Response: "A JSON object holding all infos about the genre"

  • Return data about a director (bio, birth year, death year) by name

    URL: "/movies/directors/:directorName"

    Method: GET

    Query parameters: "directorName"

    Request: "None"

    Response: "A JSON object holding all data about the searched director"

  • Allow new users to register

    URL: "/users"

    Method: POST

    Query parameters: "None"

    Request: "A JSON object holding data about the user to add. Structured like: { name: "Lulu", favoriteMovies: [], } "

    Response: "A JSON object holding data about the user that was added, including an ID: { id: 1, name: "Lulu", favoriteMovies: [], } "

  • Allow users to update their user info (username)

    URL: "/users/:id"

    Method: PUT

    Query parameters: "id"

    Request: "None"

    Response: "A JSON object holding data about the user that was updated"

  • Allow users to add a movie to their list of favorites (showing only a text that a movie has been added—more on this later)

    URL: "/users/:id/:movieTitle"

    Method: POST

    Query parameters: "id, movieTitle"

    Request: "None"

    Response: "A text message indicating whether the movie was seccessfully added"

  • Allow users to remove a movie from their list of favorites (showing only a text that a movie has been removed—more on this later)

    URL: "/users/:id/:movieTitle"

    Method: DELETE

    Query parameters: "id, movieTitle"

    Request: "None"

    Response: "A text message indicating whether the movie was seccessfully removed"

  • Allow existing users to deregister (showing only a text that a user email has been removed—more on this later)

    URL: "/users/:id"

    Method: DELETE

    Query parameters: "id"

    Request: "None"

    Response: "A text message indicating whether the user was successfully removed"
